# Online Consultation System

## Описание проекта

**Online Consultation System** — это веб-приложение для онлайн-консультаций и системы вопросов-ответов. Проект позволяет пользователям регистрироваться, входить в систему, создавать вопросы, просматривать свои вопросы, открывать страницу конкретного вопроса, редактировать вопрос, публиковать ответы и управлять личным профилем.

Проект разработан в рамках курсовой работы по дисциплине **ПМ04 «Проектирование и обеспечение бесперебойной работы web-сайта»**.

---

## Цель проекта

Цель проекта — разработать клиент-серверное веб-приложение для публикации вопросов и получения ответов в удобном и современном интерфейсе с использованием React, Node.js, Express и MongoDB.

---

## Основные возможности

- регистрация нового пользователя;
- вход в систему;
- защита закрытых маршрутов;
- создание нового вопроса;
- просмотр всех вопросов;
- просмотр только своих вопросов;
- просмотр детальной страницы вопроса;
- редактирование вопроса;
- публикация ответов;
- управление профилем пользователя;
- обработка несуществующих маршрутов через страницу 404.

## Описание клиентской части

### `components/`
Содержит переиспользуемые компоненты интерфейса. В проекте используется `ProtectedRoute.jsx` для защиты приватных страниц.

### `context/`
Содержит `AuthContext.jsx`, который отвечает за хранение токена, состояния авторизации и данных текущего пользователя.

### `layouts/`
Содержит `MainLayout.jsx` — общий шаблон страниц сайта.

### `pages/`
Содержит страницы приложения:
- `HomePage.jsx` — главная страница;
- `RegisterPage.jsx` — регистрация;
- `LoginPage.jsx` — вход;
- `ProfilePage.jsx` — профиль пользователя;
- `CreateQuestionPage.jsx` — создание вопроса;
- `EditQuestionPage.jsx` — редактирование вопроса;
- `QuestionPage.jsx` — страница одного вопроса;
- `MyQuestionsPage.jsx` — список собственных вопросов;
- `NotFoundPage.jsx` — страница 404.

### `services/`
Содержит функции для отправки запросов на сервер:
- `authService.js` — запросы авторизации;
- `questionService.js` — запросы для работы с вопросами.

---

## Описание серверной части

### `controllers/`
Содержит бизнес-логику обработки запросов:
- `authController.js` — регистрация и вход;
- `questionController.js` — создание, получение, редактирование и удаление вопросов;
- `answerController.js` — добавление и получение ответов.

### `middleware/`
Содержит `authMiddleware.js`, который проверяет JWT-токен и защищает приватные маршруты.

### `models/`
Содержит модели MongoDB, созданные через Mongoose:
- `User.js` — пользователь;
- `Question.js` — вопрос;
- `Answer.js` — ответ.

### `routes/`
Содержит маршруты API:
- `authRoutes.js` — маршруты авторизации;
- `questionRoutes.js` — маршруты вопросов;
- `answerRoutes.js` — маршруты ответов.

### `utils/`
Содержит вспомогательные функции. В проекте используется `generateToken.js` для генерации JWT.

---

## База данных

В проекте используется **MongoDB**. Подключение к базе данных и работа с коллекциями выполняются через **Mongoose**.

В базе данных хранятся:
- данные пользователей;
- вопросы;
- ответы.

## Безопасность

В проекте реализованы базовые механизмы безопасности:

- пароли хранятся в зашифрованном виде;
- для хеширования используется `bcryptjs`;
- для аутентификации используется JWT;
- приватные маршруты защищены middleware;
- пользователь не может получить доступ к закрытым разделам без авторизации.

## Установка и запуск проекта

### 1. Клонирование проекта

```bash
git clone https://github.com/your-username/online-consultation-system.git
cd online-consultation-system
```

### 2. Установка зависимостей клиента

```bash
cd client
npm install
```

### 3. Установка зависимостей сервера

```bash
cd ../server
npm install
```

### 4. Настройка переменных окружения

Создайте файл `.env` в папке `server`:

```env
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_secret_key
```

### 5. Запуск сервера

```bash
cd server
npm run dev
```

Сервер будет доступен по адресу:

```bash
http://localhost:5000
```

### 6. Запуск клиента

```bash
cd client
npm run dev
```

Клиент будет доступен по адресу:

```bash
http://localhost:5173
```

## Основные API маршруты

### Auth
- `POST /api/auth/register` — регистрация пользователя;
- `POST /api/auth/login` — вход пользователя.

### Questions
- `GET /api/questions` — получить все вопросы;
- `GET /api/questions/:id` — получить один вопрос;
- `POST /api/questions` — создать вопрос;
- `PUT /api/questions/:id` — обновить вопрос;
- `DELETE /api/questions/:id` — удалить вопрос.

### Answers
- `POST /api/answers` — добавить ответ;
- `GET /api/answers/question/:questionId` — получить ответы для вопроса;
- `DELETE /api/answers/:id` — удалить ответ.

> Если в коде часть маршрутов названа иначе, в документации нужно оставить реальные имена из файлов `routes/`.

## Логика работы приложения

1. Пользователь открывает сайт.
2. Регистрируется или входит в систему.
3. После успешного входа получает доступ к защищённым страницам.
4. Пользователь может создать новый вопрос.
5. Вопрос сохраняется в базе данных MongoDB.
6. Другие пользователи могут открыть страницу вопроса и оставить ответ.
7. Пользователь может открыть страницу своих вопросов и управлять ими.
